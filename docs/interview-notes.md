# 使用者訪談紀錄

> 這份文件可以由 Codex 先產生草稿，但你必須刪除、修改或補充。請不要直接把 AI 輸出當成最後結論。

## 我的 Phase 0 prototype 現況

目前做出的 Phase 0 原型具備了「數據儀表板」、「原始資訊」與「整理工作台」三個主分頁。原始資訊列表中展示了 12 筆帶有資訊取得方式與查核狀態的通報卡片；整理工作台則提供對每筆通報的草稿編輯器（CRUD），支援透過模擬的 AI 分析功能，為草稿補上可能類型、信心程度、下一步建議、阻礙點與備註等資訊，但在 UI 上對安全預設邊界及不確定性的標示仍較為粗糙。

## Persona 1：回報者

### 我想完成的事

- 當我和鄰居遇到狀況，或聽到一些消息時，能以最快速度把我知道的片斷線索（例如「大進路口往溪邊第二排長者需要搬家具」，或是「聽說溪畔活動中心還有雨鞋」）送出去，不用花時間整理。
- 讓我送出的資訊能被後面的人看到，但希望他們能一眼看出這是我「親眼所見」還是「聽人轉述」的，避免造成誤會。

### 我看不懂的地方

- 整理工作台上的「候選類型」（求助候選、地點狀態候選、任務候選）到底是什麼意思？我只是來回報狀況的，我不知道這在你們系統裡算哪一種。
- 還有「信心程度」和「阻礙點」，這像是要我給自己的回報打分數。如果我只是聽別人說的（比如 M-004 群組傳言），我不確定該選「低」還是「中」，如果不填會不會就不能送出？

### 我最怕的錯誤

- 我只是在群組看到一張沒日期的道路封閉截圖（M-005）並順手上傳，結果系統或 AI 卻自作主張幫我補上今天日期，甚至直接當成「官方公告」發布，導致大家走錯路。
- 志工幫不方便用手機的鄰居長輩代通報（M-011），長輩還沒有同意公開資訊，系統卻直接把「大進路口往溪邊第二排住家」這種隱私資訊大喇喇地秀在公用畫面上，甚至自動地圖定位。

### 我不敢直接相信的資訊

- 畫面上那些綠色標籤寫著「已整理」或看起來很正式的內容。比如「早上還有雨鞋」（M-002），現在都下午了，如果系統上沒有顯示最後確認的時間，我根本不敢相信現在去還有沒有。
- 那些沒有明確來源、只有一句「有人在群組說...」（M-004）的資訊，在沒有人去現場核對之前，我不敢隨便相信。

### 我希望系統提醒我的事

- 當我填寫回報時，如果地點寫得很模糊（例如「老雜貨店後面」），請提醒我「因為地點不夠具體，後續可能無法派人協助」，但不要強迫我一定要輸入精確地址才讓我送出。
- 提醒我「這筆資訊是多久以前更新的」，以及「這不是親眼看見的轉述消息」，讓我決定要不要自己去求證。

### 我想反問開發者的一個問題

- 如果我手邊只有一句「聽說老街那邊比較需要水電」（M-003），沒有任何聯絡電話、也沒有確切地址，那我到底該不該把這條不完整的資訊報上來？你們會怎麼處理它？

## Persona 2：資訊整理者

### 我想完成的事

- 將現場混亂、真偽未辨的災害通報（例如 M-004 溪畔活動中心雨鞋、M-005 中午道路封閉截圖）進行人工初步篩選，標出不確定性，整理成後續志工或決策人員看得懂的「草稿」，降低他們誤判現場情況的風險。
- 留下我們團隊在整理過程中的判斷理由與人工審查備註，確保後續接手的人知道哪些資訊是原文有的，哪些是我們的推測或人工查核結果。

### 我看不懂的地方

- 為什麼畫面的「數據儀表板」中，直接把「資訊來源類型（如 `social_post`、`official_notice`）」拿來當成資料可信度的統計指標？這會讓下一位協作者誤以為 `official_notice` 就一定是真的，即使那張截圖可能已經過期（M-005）。
- 在「原始資訊」列表和「整理工作台」中，不同資訊的 `updatedAt` 時間很混亂。我不懂這個時間代表的是「貼文發布時間」、「事件發生時間」、「我們收到這條消息的時間」，還是「最後編輯草稿的時間」？這會嚴重影響我們對時效性的判定。

### 我最怕的錯誤

- 下一線的清淤志工或物資人員，因為看了我整理完的畫面，誤以為所有資訊都已經確認，而直接跑去「老雜貨店後面」（M-001）幫忙，結果那邊根本是一天前的舊消息，或是根本找不到具體地址，浪費了寶貴的救災人力。
- AI 智慧分析功能因為「腦補」過度，自動把模糊的地址或無時間戳記的流言（例如 M-004、M-005）標記為「高品質、可直接轉成任務」，而我在批次處理時沒注意就直接儲存，導致錯誤資訊被發布出去。

### 我不敢直接相信的資訊

- 社群貼文上寫著「大進路口往溪邊方向第二排住家」（M-011）或「老雜貨店後面」（M-001）等模糊的文字敘述，在沒有與當事人本人或現場志工以電話/親自確認並取得確切經緯度前，我完全不敢直接相信。
- 外地家屬代通報親友需要藥品（M-012）這類第三方轉述資訊。我無法確認親友目前的真實位置，也無法確定他們本人是否真的需要這項協助或是否同意公開個資。

### 我希望系統提醒我的事

- 當某一筆資訊（例如 M-006 關於側門是否淹水的爭議）在不同來源中出現嚴重衝突或相反的內容時，系統應該用醒目的方式（如警示框）提醒我注意衝突，而不是只顯示其中一項。
- 當我正在編輯的草稿中包含可能涉及長者個資、隱私（例如 M-011 志工代轉述）或電話號碼時，系統應該主動提醒我確認「當事人是否同意公開完整資訊」，防止個人隱私在未經授權下流出。

### 我想反問開發者的一個問題

- 請問目前的「整理工作台」上，既然說這只是個「草稿」，那當我按下「儲存草稿」後，這些資料到底會流向哪裡？後續的其他現場志工真的會直接看到這些草稿嗎？還是會有另一個審核發布的流程？

## Persona 3：行動者

### 我想完成的事

- 我想要快速從清單中挑出「現在安全、可以直接出發」的任務，例如去哪裡幫忙清淤、帶什麼工具去。
- 我需要確認現場物資的即時狀態（例如哪裡的雨鞋還有剩、哪裡已經不缺了），避免帶了沒用的東西或讓大家白跑一趟。

### 我看不懂的地方

- 為什麼左邊清單有些通報會顯示綠色的「已整理」標籤，但點進去卻在卡片上看到「不可直接行動」？「已整理」到底代表這件事是可以去了，還是只代表有人填過表單？
- 數據儀表板上的「待確認」和「需人工確認」有什麼不同？我身為要出發的人，到底該看哪一個狀態來決定能不能行動？

### 我最怕的錯誤

- 看到寫著「已整理」，以為是安全任務，大老遠跑到現場，才發現只是沒有日期戳記的社群流言（例如 M-004、M-005），根本沒有雨鞋，或者路是封閉的。
- 跑去一個模糊的地址（例如 M-001 「老雜貨店後面」或 M-011 「大進路口往溪邊第二排住家」），在現場挨家挨戶找人，不但找不到人，還打擾到不需要幫助的居民。

### 我不敢直接相信的資訊

- 那些寫著「AI 智慧分析」的信心程度和分類（例如把沒時間戳記的截圖標為公告）。我怕 AI 為了填滿格子，就把「可能」腦補成「確定」。
- 隔了幾小時以上的物資回報（...），這很可能早就被拿光了，但系統上依然顯示存在。

### 我希望系統提醒我的事

- 只要這筆通報沒有明確的「現場志工確認時間」或「查證人」，就應該用刺眼的黃色或紅色警告我「此資訊未經實地確認，請勿出發」。
- 當有多個通報互相矛盾時（例如 M-006 有人說學校側門可當集合點，有人說淹水了），在畫面最上方同時跳出衝突警示，阻止我前往。

### 我想反問開發者的一個問題

- 當我人在現場，發現實際狀況和系統寫的不一樣（例如 M-010 現場發現雨鞋已經發完了），我要怎麼在畫面上立刻回報「這筆資訊已經過期」，好讓其他志工不要再跑一趟？

## 人類補充

請寫下 Codex 沒有注意到，但你覺得重要的觀察。

- **三類使用者的角色介面混亂**：目前的單一 App 介面把回報、整理與行動混在一起，回報者看到複雜的整理欄位心生畏懼，行動者看到「已整理」誤以為是已核實任務，這反映出單一入口對不同角色的混淆。
- **過度信賴資料來源類型**：系統無意間以 `sourceType` (社群或官方) 作為信任基礎，忽略了官方截圖亦會過期（時效性），或社群貼文亦可能有高品質回報（如 M-010），應該將資訊來源與查證狀態徹底解耦。
- **缺乏狀態流轉生命週期**：資料只有 `is_checked` 的二元狀態，無法呈現「未整理 -> 整理中 -> 需人工複核 -> 已發布任務 -> 已過期」的流轉生命週期，導致各角色無從掌握資料的確定性階段。
